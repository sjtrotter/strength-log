Product & design brief · for the design pass · 2026-07

strength.log

A no-frills, local-first strength-training tracker for Android and Wear OS. One-time purchase, no account, no ads, no network permission — the app cannot phone home. A lifter answers a short wizard once, then lives on a single workout screen where every tap is cheap.

Native Android · Jetpack Compose Wear OS companion Dark only (v1) Offline by construction Phone-sized, portrait

This page is set in the app's own tokens — near-black ground, the four earth-tone day accents (section markers cycle A→D exactly as the app cycles day colors), and its bundled Oswald for numerals and labels. Reading it is a taste of the product.

A

The user, and where the app gets used

The reference user is a ~40-year-old training for maintenance and health, not personal records. Their schedule is unreliable — sessions get missed and the plan must absorb that gracefully. They want a good pump on weights they can own, balanced development, and honest logging of what actually happened.

Context of use is the gym floor: phone on a bench or the floor, sweaty or chalky hands, 30-second glances between sets under bad lighting. Design implications, already load-bearing:

  • Glanceable numerals. The weight is the hero of every row — big, condensed, high-contrast.
  • Cheap taps. Steppers, never keyboards. Checkmarks, never forms. Touch targets ≥ 48dp (enforced in code).
  • Cards collapse when done so the screen always answers "what's next?", not "what did I do?"
  • A keep-screen-on toggle lives on the workout screen.
B

Four ideas the design must communicate

1 · Rotation, not calendar

Workout days are lettered A, B, C, D — never Monday/Tuesday. Finishing a workout advances the rotation (A→B→…→A). Missing a day shifts the plan; it never skips a muscle. The app suggests the next day but the user can override by tapping any tab — an override is visibly flagged, never punished. There are no streaks, no guilt mechanics anywhere.

2 · GOAL vs ACTUAL

Every exercise shows a calculated, read-only GOAL — the maintenance weight derived from bodyweight, age, experience, and goal emphasis. The editable log is seeded from the GOAL once, then becomes the lifter's living record. The two must read as different classes of information: GOAL is reference (calm, fixed, accented); the log is theirs (interactive, persistent).

3 · The top set is the anchor

Main lifts are a Madcow-style sequence: four ramp sets (55/70/80/90%, the last a lighter triple) → the TOP set → a back-off set. Editing the TOP set's weight recalculates the whole ramp and back-off (the "cascade"). Editing any other set changes only that set. The TOP row is visually distinct (accent border) because it is mechanically special.

4 · Every tap is cheap

The entire in-workout interaction vocabulary is: tap +/−, tap a checkmark, tap a header to collapse, tap DONE. Anything requiring typing (custom exercise names) happens outside the workout.

Complete behavior inventory

  • Tab strip: ⚙ + one lettered tab per day, each tinted with its day accent. The suggested-next day carries a dot + outline. Selecting a non-suggested day shows the override line: "Viewing Day X (override). Suggested next: Day Y."
  • Card header: name · MAIN badge (mains) · "+1 WARM-UP" badge (first new-pattern accessory of the day) · green ✓ chip when all sets done · GOAL block top-right (tiny "GOAL" label over a large accent numeral, "/hand" suffix for dumbbell-style lifts). Supersets title as "SS: {Primary} + {Partner}".
  • Mains helper line: "change the TOP set → ramp & back-off recalculate."
  • Set rows: kind label (R1–R4 · TOP accented · B/O · plain numbers for regular/extra sets) · weight stepper · reps stepper · ✓ · remove. Steppers move by 5 lb / 2.5 kg (2.5 lb / 1.25 kg under 20 lb); reps by 1. Never below 1 set.
  • Cascade: TOP-weight edits rewrite R1–R4 and B/O instantly. This deserves a designed micro-moment — right now the numbers just flicker into place.
  • Supersets: partner sub-row per primary row — indented, ↳ marker, dashed top border, muted ground, own steppers. One ✓ per round. Add/remove keeps both tracks aligned.
  • Collapse: auto when every set is ticked (green left edge + ✓ chip); header tap toggles manually and overrides auto. Summary line when collapsed: 185×8 · 185×8 · 185×7, supersets 60×12(50×15) / 60×11(50×14), or 3 sets · GOAL 90 if nothing is ticked yet.
  • Checkmarks reset per calendar date — every new day starts fully expanded and unticked; weights and reps persist as "last time".
  • Cardio finisher card below the list (collapsible, default closed): hard intervals only after leg-light lifting, easy Zone 2 after leg-heavy days. Copy reminds: lift first, then run.
  • DONE — ADVANCE TO DAY {next}: the one big button. Writes an immutable session record, clears today's ticks, advances the rotation.
  • Footer: program-philosophy blurb · "clear today's checkmarks" · keep-screen-on toggle.
  • Planned (data already exists): a subtle per-exercise "last time: 185×8" sourced from history.

B

The other screens, briefly

Wizard (6 steps + optional equipment step)

Training goal → days/week (2–6) → split (sane default per days/week, e.g. full-body rotation at 4) → main-lift anchors (three named presets; deadlift style choice) → cardio prefs → about you (bodyweight/age/level). Every step pre-answered — the design should celebrate that "just keep hitting next" is a first-class path, not a skipped configuration. Ends by generating the program.

Setup

The same inputs, editable anytime, with a live preview of the four main-lift GOALs recalculating as the steppers move — the one place the goal math becomes visible and tangible. Re-run-wizard sits here behind a destructive confirm.

Day edit sheet

Swap opens "Best replacements" — same-movement-pattern candidates in ranked order, searchable and filterable by equipment. Old logs are kept; the new exercise seeds fresh from its own GOAL. Add picks pattern → exercise; remove enforces a 3-exercise minimum; "reset day to template" regenerates from stored wizard answers.

Log

A plain reverse-chronological list of completed sessions. It's the honest record, the export source, and where other apps' workouts surface — not an analytics dashboard. Charts are deliberately v2.

Watch

Two levels: today's exercise list → one exercise with GOAL and set rows, rotary or ± input, one tick per set/round. Everything oversized and glanceable mid-set; the phone remains the source of truth.

D

Constraints vs freedom

Pinned — do not redesign
  • Dark-only in v1; near-black ground + the six spec hexes (bg/surface/border + four accents)
  • Per-day accent identity and its A→D cycle
  • Steppers, not keyboards; checkmarks, not forms
  • Touch targets ≥ 48dp; text-on-accent contrast ≥ 4.5:1 (both enforced by tests)
  • All §8.2 behaviors above (cascade, one-tick rounds, collapse rules, per-date reset, override note)
  • GOAL is read-only, always
  • Condensed display numerals as the typographic signature
  • No purple, no default-Material sameness, no emoji in UI, no wall-of-cards genericism (house rule)
  • Platform: Compose Material 3 components under a full custom theme; portrait phone + watch
Open — design owns this
  • Layout, density and rhythm of the day screen — especially the set-row grid (current build stacks weight over reps; a tighter single-line row is likely possible at gym-glance sizes)
  • Tab strip treatment, selected/suggested/override states
  • The cascade micro-moment; collapse/expand motion; the DONE moment
  • Derived tones (text, muted, done-green, error, container fills) — recolor freely if contrast tests are updated with intent
  • Iconography (currently near none — text glyphs)
  • Wizard personality, empty/first-run states, destructive confirms
  • History list, substitution picker, setup preview presentation
  • Watch UI wholesale (only the one-tick-per-round rule is fixed)
  • Type scale extensions, spacing system, corner radii, elevation language (currently flat + hairline borders)

Explicitly out of scope for v1

History charts · plate calculator · rest timers · RPE · cloud sync/accounts · light theme · tablet layouts · social anything. Don't design for them; v2 may add charts, a widget, and Drive backup.

A

Where the design pass lands

Build state: domain math, storage, the token layer, and a functional day screen exist; wizard, day-edit, setup, log, custom-exercise, export, Health Connect, and the watch are still to be built and can adopt a new direction from day one. The day screen will be restyled to match.

The highest-value design targets, in order:

  • The set row. The single most-repeated element in the product. It must fit weight stepper + reps stepper + tick + kind label in one comfortable line at ≥48dp targets, readable at arm's length.
  • The exercise card system — expanded, superset, collapsed, done — as one family with a clear state language.
  • The day header — tabs, override note, day title + emphasis line — where rotation-not-calendar either clicks or doesn't.
  • The wizard — six screens that set the tone of the whole product in the first two minutes.
  • The watch — the purest expression of "glanceable logging".

Companion artifacts for a working session: the product spec (STRENGTH_TRACKER_SPEC.md) pins every behavior; docs/PLAN.md lists the amendments (history, custom exercises, kg, equipment filter). This brief is the design-facing digest of both.
